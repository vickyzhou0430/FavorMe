<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Backend error body to UI | Server-provided error fields influence user-visible error state |
| User input preservation | Raw question remains editable after invalid input |
| Retry action to API client | User-triggered retries repeat prior network requests |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03-01 | Information Disclosure | `insight_view_model.dart` | mitigate | Preserve raw question in memory for retry/editing only; do not log raw question bodies or include them in exception text. |
| T-02-03-02 | Denial of Service | `insight_view_model.dart` | mitigate | Disable send/option/retry while the matching async request is in flight and assert duplicate taps make one API call. |
| T-02-03-03 | Spoofing | `insight_api_client.dart` | mitigate | Continue requiring configured bearer token and stable device id on retries; retry must reuse the same auth/header path. |
| T-02-03-04 | Tampering | `InsightApiException` parsing | mitigate | Treat malformed error bodies as generic typed errors and render safe local copy instead of trusting arbitrary backend copy blindly. |
</threat_model>
