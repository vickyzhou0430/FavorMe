# ADR-002：后端技术栈

- **状态**：已接受
- **日期**：2026-04-24
- **决策者**：项目维护者

## 背景

FavorMe 需要**可长期演进**的后端：用户与资料、收藏、AI 网关、配额、后续订单/权益预留，并能在 **7 月首版**前与 `WebView` 壳 + H5 及后续 **Flutter** 客户端稳定对接。

## 决策

采用以下**主栈**（代码目录 `backend/`，待初始化）：

1. **TypeScript** + **NestJS** 作为 HTTP / 业务与 AI 网关服务框架  
2. **PostgreSQL** 作为**主**关系型数据库  
3. **Prisma** 作为 ORM  
4. **Redis** 在 **MVP 即**引入，用于限流、配额、会话/临时态等  
5. **REST + JSON** 为默认对外 API 形态

大模型经 **统一 AI 网关** 在服务端发起；**开发期**优先**火山** API 联调，**允许**境外 provider，不将「数据不出境」立为**唯一**技术约束（上线前法务与合同**单独**评审）。

## 备选方案

- **Java / Spring Boot**  
  企业成熟、招聘池大；单人 **MVP** 迭代与**与 Next/H5 的 TS 生态**协同略重，**暂缓**为首选。  
- **Python / FastAPI 作主后端**  
  适合**算法/数据侧**中台，未来若需独立**检索/批处理/实验服务**可**旁路**部署，不替代主产品 API 栈。  
- **MongoDB 作主库**  
  对「订单/关系/强一致」的后续扩展不友好，**不采用**作核心事务库。

## 后果

- **正面**：与 `demo/web` 同语言、模块边界清晰、**Prisma** 迁移与类型联动顺、**Redis** 在防刷与体验上**不必二次迁移**。  
- **风险**：需掌握 **Prisma 迁移/部署** 与云托管 **PG/Redis** 的备份策略；`NestJS` 有学习曲线但**长期可维护**。  
- **待办**：在 `docs/modules/auth.md`、`ai-gateway.md` 中接口与表结稿；`backend` 初始化任务进入 `docs/tasks/001-*.md`（下一条任务）。
