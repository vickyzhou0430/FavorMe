> **状态：已更新（会议纪要同步）**（2026-05-09，与 [`architecture.md`](architecture.md)、[`tech-stack.md`](tech-stack.md) 同步）
>
> **⚠️ 主线已升级到「点醒 · 动态问卷」（PRD V2.0，2026-06-13）**：核心链路从「固定三问」升级为**有状态动态问卷**——逐题实时生成、**动态 3–5 道收题**、**五级量表**作答、结构化「点醒报告」（金句 / 倾向 / 分析 / 轨迹 / 行动建议）、会话落库与历史复盘。权威实现与契约见 [`modules/insight-v2-dynamic-questionnaire.md`](modules/insight-v2-dynamic-questionnaire.md) 与 [ADR-005](decisions/005-dynamic-questionnaire-state-machine.md)。下文「三问」描述为历史阶段，作为背景保留。

## 1. 一句话

FavorMe 当前聚焦为「**AI 心理洞察助手**」：不替用户做决定、不输出外部标准答案，而是通过结构化三问帮助用户看清内在倾向，找到那 `51%` 的真实选择。

## 2. 产品定位与核心价值

- 从「玄学决策」转向「**心理洞察 + 轻决策推动**」。
- 核心价值是通过高质量追问引导用户自我探索，最终起到“推一把”的确认作用。
- 与通用聊天 AI 的差异：聚焦生活中的“小而即时纠结”，以三问快速获得倾向，而非长对话和参数分析。

## 3. MVP 核心链路（当前第一优先级）

1. 用户登录（手机号 / 微信 / Apple ID 等可扩展）  
2. 个人中心填写基础画像（账号名、性别、头像、出生日期、MBTI 等）  
3. 在首页输入问题（文字输入为主，兼容语音转文字）  
4. 可选补充背景（可跳过）  
5. 系统生成 3 个关键问题（带选项）  
6. 用户用左/右滑或点击选项快速作答  
7. 输出倾向性结论（轻量反馈），完成后可回到首页继续使用

> MVP 先打通 `用户提问 -> 生成三问 -> 用户作答` 最短链路，其他体验优化后置。

## 4. 三问方法论（问题生成约束）

系统生成问题必须遵守明确框架，至少覆盖以下三层剖析目标：

1. **剥离外界干扰**：识别用户真实偏好，而非“别人希望我怎么选”。  
2. **直面恐惧底线**：帮助用户确认最坏情况中自己真正不能接受的是什么。  
3. **区分主动与逃避**：判断当前倾向是主动追求，还是仅为逃离现状。  

补充约束：
- 有背景信息时，问题需更具体、贴合情境。
- 无背景信息时，使用通用但有效的本质剖析问题集。
- 三问交互为问卷式快答，不演化为冗长对话流。

## 5. 输入与边界处理

- 支持文字输入与语音转文字输入并存。
- 对明显异常输入（如非问题句、无语义噪声）进行拦截提示，不强行生成三问。
- 不做复杂兜底对话；优先保证主链路稳定、清晰、可复用。

## 6. 商业化与权益策略（预留）

- 可采用分层输出：基础倾向免费，完整解析可挂会员权益。
- MVP 阶段可先不闭环支付，但需在数据模型中预留权益扩展位。

## 7. 客户端与发布策略（试水阶段）

| 阶段 | 形态 | 说明 |
|------|------|------|
| 当前试水 MVP | **Android 原生 App 优先** | 以最快速度验证核心三问链路与留存反馈 |
| 试水后扩展 | iOS / WebView / 小程序按验证结果再决策 | 小程序是否满足交互与 chat 能力需单独调研 |

**系统版本策略**：优先覆盖 Android 近 2 个主要大版本，具体 `minSdk` 在客户端工程初始化时落地。

## 8. 明确不做的方向（除非决策变更）

- 医疗诊断、精神治疗替代类承诺。
- 以算命断言、恐吓、迷信话术驱动的高风险表达。
- 在客户端长期明文持有厂商密钥与完整商用 prompt 策略。
