# 部署与环境

> 状态：**骨架**。在选定云厂商与首版环境后由负责人补全；敏感信息**不得**提交到仓库。

## 1. 环境划分（建议）

| 环境 | 用途 | 说明 |
|------|------|------|
| `dev` | 本地 + 个人联调 | 可连沙箱模型、脱敏数据 |
| `staging` | 与线上同构预发 | 发版前回归、联调 App |
| `prod` | 生产 | 正式用户、监控与备份 |

TBD：是否首版就三环境，或先 `dev` + `prod`。

## 2. 配置与密钥

- 使用云厂商 **密钥管理** / **环境变量注入**，不将 Key 写入客户端与镜像层。
- `backend`：数据库 URL、Redis、对象存储、模型 API Key、短信/登录等，分环境配置。
- 旋转策略与负责人：TBD。

## 3. 可观测性（目标）

- 应用日志：结构化，可检索
- 错误聚合：如 Sentry 或云错误诊断
- 基础指标：QPS、延迟、5xx
- 业务事件：注册、关键 AI 调用结果（注意脱敏）

TBD：具体产品选型。

## 4. 静态与客户端

- `demo/web` 出包：静态资源 + Node SSR 或纯静态，视部署方式定。
- App：走应用商店与各自签名流程，与本文「服务端部署」独立但需同一 `API Base URL` 配置策略。

## 5. 检查清单（发版前）

- [ ] 环境变量与密钥已注入、未进 Git
- [ ] 数据库备份策略

- [ ] 与法务/备案相关项（若适用）
