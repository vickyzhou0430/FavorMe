## 一、产品定位与核心理念

在当今信息过载的时代，人们在面临选择时往往不是因为缺乏信息，而是陷入了「决策疲劳」与「内心冲突」（Ambivalence）。正如心理学中的动机访谈（Motivational Interviewing）理论所指出，人们在面对改变或选择时产生的纠结是正常的心理过程[^1]。

本产品定位为一款**基于心理学理论的 AI 决策辅助工具**。

- **核心理念**：真正的决策不是被外部信息说服，而是被自己内心的真实偏好所「点醒」。产品不替用户做决定（例如直接告诉用户选 A 还是选 B），而是扮演一面「心理镜子」，通过动态生成的测试题，帮助用户拨开迷雾，看见自己潜意识里早已做出的选择。
- **产品形态**：从传统的「自由对话流」升级为「动态问卷（Dynamic Questionnaire）」。这种形态能显著降低用户在纠结时的表达压力，强化「测试」的客观仪式感，并有效约束 AI 输出，确保整个流程沿着「探索—挖掘—点醒」的既定轨道前进。
- **设计风格**：**深夜诊室（Dark Therapy Room）**。极简深色布局，琥珀金 `#E8A84C` 作为唯一强调色，文字优先的信息层级，移动端单列流，打字机动效，品牌语气聚焦于「照见自己」而非「给出建议」。

## 二、全局架构与交互流程

整个产品的交互链路分为三个阶段：

### 1. 倾诉与破冰（入口层）

用户进入首页后，看到一个全屏大输入框，配有「常见纠结」预设场景卡片（当前已实现：辞职创业、两个 offer、感情关系、出国读研、消费决策、主动联系等 10 余个场景）。用户可自由描述纠结，也可点击预设场景快速填入。

AI 的处理逻辑如下：

| 输入状态 | AI 行为 |
|----------|---------|
| 能识别出两个选项 | 直接进入动态问卷（出第 1 题） |
| 有方向但信息不足 | 生成 1 个开放式追问（最多追问 2 次） |
| 完全没有方向 | 强制追问，2 次后仍不足则强制出题 |

### 2. 动态测试（核心层）

**题目数量：固定 5 道，不多不少。** 这是当前实现与原方案的核心差异——原方案设计为「3–5 道动态收题」，实际工程实现中为了保证体验一致性和 Prompt 可控性，改为固定 5 道。

**选项机制：五级量表。** 每道题除 A / B 两个极端立场外，用户还可以选择「偏 A」「偏 B」「中间」，共五档。这一设计解决了原方案二选一时用户「两个都有点像」的困境，同时保留了足够的区分度供 AI 分析。

| 选项 | 含义 |
|------|------|
| A | 完全认同选项 A 的立场 |
| 偏 A | 更倾向 A，但不完全确定 |
| 中间 | 两边都有共鸣，难以明确偏向 |
| 偏 B | 更倾向 B，但不完全确定 |
| B | 完全认同选项 B 的立场 |

- **进度显示**：界面使用圆点进度指示器（共 5 个点），不显示数字编号，保持测试的神秘感。加载时显示「正在深入你的潜意识……」动效。
- **换一题（规划中）**：每道题下方提供「换一题」按钮，点击后 AI 使用不同策略重新出题，不计入已答题数，5 题总量不变。

### 3. 点醒与呈现（结果层）

结果报告页层次分明，内容结构如下：

1. **点醒金句**（大字，打字机逐字呈现）：一句直击灵魂的话，指出用户隐秘的恐惧或真实的渴望。金句标准：把这句话发给其他人，对方会说「这不是说我的」——证明它是独特的，而非通用套话。
2. **内心真实倾向**（15 字以内的标签）：用简短词语点明用户的潜意识倾向方向。
3. **潜意识分析**（100–200 字）：必须引用用户在问卷中的具体选择作为证据，给出心理学解释，指向用户没有意识到的潜意识层面。
4. **选择轨迹回顾**：展示用户每道题的选择（含五级程度和策略标签），供用户复盘。
5. **行动建议**（30–50 字）：一个具体的、今天就能做的小行动或微小实验，而非宏大的人生建议。

**分享卡片（规划中）**：报告页提供「分享金句卡片」按钮，生成包含金句、内心倾向和品牌标识的精美深色卡片，支持保存图片或系统原生分享。

## 三、提问策略库（8 种核心策略）

为了让产品不仅能处理「人生重大决策」（如职业、买房），还能完美适配「日常小纠结」（如消费、社交回复、习惯养成），AI 必须从以下 8 种策略中选择生成题目，且**不能连续使用同一种策略**。5 道题须覆盖不同维度：建议包含至少 1 道损失厌恶/极端情境类（情绪反应）、1 道行为核查类（实际行动）、1 道价值观/遗憾类（深层动机）。

### （一）探测「真实意愿」——适用于：用户声称「都挺好」或「都想要」

#### 策略 1：损失厌恶测试（Loss Aversion Test）

该策略基于行为经济学中的「损失厌恶」理论，即人们对失去某物的痛苦感受，远大于获得同等事物的快乐感受[^2]。核心逻辑是假定某个选项突然不可用，测试用户的本能反应是「松了一口气」还是「感到遗憾」。适合面对多个好机会的取舍（如两个 offer、两个心仪商品）。

#### 策略 2：心理距离拉伸（Construal Level Shift）

建构级别理论指出，当事物在时间或空间上离我们很近时，我们关注的是具体细节和阻碍；当距离拉远时，我们关注的是核心价值和意义[^3]。该策略通过把决策时间线拉长，或把决策主体换成别人，屏蔽当下情绪干扰。适用于陷入细节对比无法自拔、或日常消费纠结的场景。

#### 策略 3：极端情境假设（Extreme Scenario Simulation）

认知重构通过改变问题的前提条件，帮助个体识别出阻碍行动的非理性恐惧[^4]。该策略移除现实中最关键的阻碍因素（如钱、面子、失败风险），测试用户的纯粹倾向。当用户受限于外部条件而不敢行动时尤为有效。每次只移除一个最关键的阻碍，不同时移除多个。

### （二）破解「知行不一」——适用于：用户表现出「想做但一直没做」

#### 策略 4：行为核查（Behavioral Verification）

人们经常存在动机盲点，高估自己的意愿，而真实的意愿总是体现在过去的行动分配上。该策略的核心是不问「想不想」，而是问过去「做了什么」。特别适用于习惯养成、学习计划以及大件消费（买来吃灰类）的纠结。

#### 策略 5：沉没成本剥离（Sunk Cost Isolation）

沉没成本谬误是指人们倾向于继续坚持一个不好的决定，仅仅因为之前已经投入了时间、金钱或情感[^5]。该策略假设过去的一切投入清零，只看未来，询问用户是否还会做同样的选择。在考虑要不要分手、放弃烂项目或退掉不喜欢的课时非常有效。

### （三）挖掘「底层动机」——适用于：外部压力与内部需求的冲突

#### 策略 6：价值观拉升 / 身份认同（Identity-Based Choice）

基于身份的动机理论认为，个体的决策往往是为了向自己和他人证明「我是谁」[^6]。该策略将具体的选择题转化为关于「自我认同」的选择题，适用于职业规划、生活方式选择和人生重大转折。

#### 策略 7：遗憾最小化框架（Regret Minimization）

这是杰夫·贝索斯著名的决策模型，基于预期遗憾理论[^7]。它通过将自己投射到生命末期，来评估当前选择的长期价值。核心逻辑是不比哪个收益大，而是比哪个错过了更后悔。适用于充满风险的尝试或打破常规的决定。

#### 策略 8：自我决定论核查（Self-Determination Check）

自我决定论指出，真正的内在动机来源于满足自主性、胜任感和归属感这三个基本心理需求[^8]。该策略用于区分想法是「自己真正想要的（自主）」还是「外界要求这么做的（受控）」。适用于迎合父母期望的决策、随大流的考研考公或身材焦虑引发的减肥。

### 策略调用指南

| 冲突类型 | 核心表现 | 优先调用的策略组合 |
|----------|----------|-------------------|
| A 还是 B（势均力敌） | 两个选项都好，怕选错 | 损失厌恶测试 + 遗憾最小化 |
| 做还是不做 | 想行动但缺乏动力或有现实顾虑 | 行为核查 + 极端情境假设 |
| 放弃还是坚持 | 舍不得过去的投入，又看不到希望 | 沉没成本剥离 + 极端情境假设 |
| 外界期待 vs 内心渴望 | 觉得「应该做」，但心里「不想做」 | 自我决定论核查 + 价值观拉升 |

## 四、动态问卷链路示例（四个真实场景）

### 示例一：工作与生活的平衡（外界期待 vs 内心渴望）

**用户输入**：我现在 25 岁，初入职场。时常纠结是该把时间更多花在工作上提升能力，还是培养业余爱好丰富人生。有时觉得人生该丰富，有时又觉得趁上升期该多拼工作。

**AI 识别**：选项 A（拼工作），选项 B（培养爱好）。冲突类型：外界期待 vs 内心渴望。

**第 1 题（策略 4：行为核查）**

AI：在过去三个月里，你实际上是怎么分配业余时间的？

- A：大部分时间还是在工作或学跟工作相关的东西，兴趣爱好偶尔才碰
- B：两边都有在做，但经常做兴趣爱好的时候会有一种「我是不是在浪费时间」的罪恶感

用户选择：A

**第 2 题（策略 3：极端情境假设）**

AI：假设有人告诉你，你所在的行业未来三年内会经历大洗牌，努力也不一定能保住上升通道。在这个前提下，你会怎么看待现在的时间分配？

- A：那我更应该趁现在拼一把，不确定性越大越要努力
- B：那我可能会重新考虑，觉得应该把一部分时间留给自己真正喜欢的事

用户选择：A

**第 3 题（策略 6：价值观拉升）**

AI：你说「人生应该丰富一点」——你觉得这句话是你真实的想法，还是你觉得「一个正常人应该这么想」？

- A：是真实的想法，我确实有一些事情想做，只是一直没时间
- B：说实话，可能更多是觉得「应该」这样，但我自己对兴趣爱好没有特别强烈的渴望

用户选择：B

（第 4、5 题继续深挖，固定出满 5 道后收题）

**结果页呈现**：

- **点醒金句**：你根本没有在纠结「工作还是兴趣」，你在纠结「我是不是一个足够好的人」。
- **潜意识分析**：你过去三个月实际上大部分时间都在工作，遇到外部不确定性你的本能反应是更拼，而且你自己承认对兴趣爱好并没有特别强烈的渴望。这说明你现在的真实倾向就是想拼工作。你脑子里有一个声音在说「正常人应该有丰富的业余生活」，你在用一个并不真实存在的渴望，来质疑一个其实已经很笃定的选择。
- **行动建议**：你不需要强迫自己培养兴趣爱好。现阶段全力投入工作是你真实的选择，不是你「没有生活」的证明。等未来某天你对某件事有了真实的、自发的渴望时，再给它时间也不晚。

### 示例二：职业选择（A 还是 B 势均力敌）

**用户输入**：字节 30k 稳定但岗位边缘，AI 创业公司 22k 但感兴趣，不知道怎么选。

（详细问卷流程同原方案，固定出满 5 道后收题）

**点醒金句**：你以为你在纠结「稳定还是冒险」，其实你早就做出决定了。

### 示例三：消费决策（买还是不买）

**用户输入**：手机拍照够用，但感觉差点意思，想买 6000 元的微单，妈妈说是冲动消费。

（详细问卷流程同原方案，固定出满 5 道后收题）

**点醒金句**：你想买相机，本质上买的是一个「更认真对待生活」的自我形象。

### 示例四：关系决策（做还是不做）

**用户输入**：两年前有点不愉快，最近偶尔想起老朋友，想联系但感觉很奇怪。

（详细问卷流程同原方案，固定出满 5 道后收题）

**点醒金句**：你在用「不知道该不该联系」来回避「我怕被冷落」这个更真实的担心。

## 五、System Prompt 与工程实现

### 核心 System Prompt（当前部署版本）

当前部署的 System Prompt 采用状态机模式，共分九节，强制输出 JSON 格式。以下为完整结构：

#### 一、输出格式规定（三种 JSON 类型）

```json
// 追问（信息不足时）
{"status":"need_info","clarify_question":"追问内容（不超过30字）"}

// 出题（信息足够时）
{"status":"questioning","question":{"question":"问题文本（口语化，不超过50字）","option_a":"选项A极端立场（10-20字）","option_b":"选项B极端立场（10-20字）","strategy":"策略名称","reasoning":"内部推理，用户不可见"}}

// 收题（已出满5道题后）
{"status":"finished","report":{"awakening_quote":"点醒金句（20-40字）","analysis":"心理学分析（100-200字）","tendency":"用户潜意识倾向（15字以内）","action_advice":"具体行动建议（30-50字）"}}
```

> **注意**：`option_a` 和 `option_b` 代表两个极端立场，用户可在 A、偏 A、中间、偏 B、B 五个程度中选择。选项文本只需描述两端的极端立场，不需要描述中间状态。

#### 二、每次回复前的强制检查（计题逻辑）

在生成任何内容之前，先数对话历史中 `"status":"questioning"` 的消息条数，记为【已出题数】：

- 已出题数 = 0：执行 Stage 1 或 Stage 2 任务
- 已出题数 = 1、2、3、4：必须继续出题，`status` 只能是 `"questioning"`，绝对禁止输出 `"finished"`
- 已出题数 = 5：必须收题，`status` 只能是 `"finished"`

题目数量固定为 5 道，不得提前收题，不得超过 5 道。这个判断是最高优先级，任何其他理由都无法覆盖它。

#### 三、Stage 1：分析初始描述（已出题数=0，已追问数=0）

识别纠结结构（A 还是 B / 做还是不做 / 放弃还是坚持 / 外界期待 vs 内心渴望），判断信息是否足够出题。信息不足则追问，信息足够则直接出第 1 题。

#### 四、Stage 2：分析补充描述（已出题数=0，已追问数≥1）

综合初始描述和补充描述重新识别纠结结构。最多追问 2 次，2 次后仍不足则强制出题。

#### 五、Stage 3：根据用户选项决定下一步（已出题数≥1）

充分利用五级程度信息（「偏 A」比「A」温和，「中间」表示两端都有共鸣）理解用户真实倾向。用户选择「中间」时，说明这道题没有明显区分度，下一题需要换更有穿透力的策略。

#### 六、冲突类型识别与策略优先级

| 冲突类型 | 推荐出题顺序 |
|----------|-------------|
| A 还是 B（势均力敌） | 第 1 题损失厌恶，第 2 题行为核查，第 3 题价值观拉升 |
| 做还是不做 | 第 1 题行为核查，第 2 题极端情境假设，第 3 题心理距离拉伸 |
| 放弃还是坚持 | 第 1 题沉没成本剥离，第 2 题极端情境假设，第 3 题价值观拉升 |
| 外界期待 vs 内心渴望 | 第 1 题自我决定论核查，第 2 题行为核查，第 3 题价值观拉升 |

#### 七、出题策略库（8 种）

每种策略包含：心理学原理、适用场景、出题思路、选项两端描述、禁止事项。策略库只是帮助理解心理学原理，问题和选项的具体表达完全由 AI 根据用户情境自由创作，禁止套用示例句式。

#### 八、出题规则

- 每次只出 1 道题，问题不超过 50 字，不把选项内容嵌入问题句子
- 选项必须口语化，是用户内心的潜台词，严禁出现「都行」「不知道」
- 两个选项必须有真实张力，不能一个明显「正确」一个明显「错误」
- 不能连续使用同一种策略
- 不要透露分析意图，不要说「根据心理学」
- 问题必须有侧面性，绝对不能直接问「你更想要哪个」

#### 九、点醒报告生成策略

金句标准：找到用户没有说出但已经在选择中流露的东西（真实动机、行动证明的倒向、真正在怕的东西、对自己的误判）。判断标准：把这句话发给其他人，对方会说「这不是说我的」——证明它是独特的，而非通用套话。

### 工程实现要点

**技术栈**：React 19 + TypeScript + Tailwind CSS 4 + tRPC 11 + Express 4 + MySQL（Drizzle ORM）

**前端状态机（`useDecisionAI` Hook）**：

```
idle → need_info → questioning → finished
```

用户选择通过五级量表传递给 LLM，格式为：

- `"我选A（完全认同）：{option_a}"`
- `"我偏A（但不完全确定）：{option_a}"`
- `"我选中间（两边都有共鸣，难以明确偏向）：A是'{option_a}'，B是'{option_b}'"`

**关键工程避坑**：

1. **强制 JSON**：API 调用时加上 `response_format: { type: "json_object" }`，并实现四重健壮解析（直接解析 → 移除控制字符 → 转义换行符 → 提取 JSON 块）。
2. **上下文回传**：用户做选择后，必须把完整的选项文本一起传回（而非只传「A」或「B」），否则 AI 会失去上下文。
3. **五级程度信息**：传递给 LLM 的用户回答必须包含程度描述，让 AI 能区分「完全认同」和「偏向」的差异。
4. **计题逻辑**：前端不做收题判断，完全由后端 System Prompt 的强制检查控制，避免前后端状态不一致。

## 六、历史记录模块

### 当前已实现（v1 版本）

- **自动保存**：每次完成测试后，系统自动将完整会话（纠结描述、追问 Q&A、5 道题的选择轨迹、最终报告）存入数据库，无需用户手动操作。
- **历史列表页（`/history`）**：展示所有历史测试记录，每条记录显示纠结描述、点醒金句、内心倾向标签、完成时间（今天/昨天/N 天前）和答题数。
- **历史详情页（`/history/:id`）**：完整复盘一次测试，包含原始纠结、追问 Q&A、每道题的选择（含五级程度和策略标签）、点醒金句、潜意识分析和行动建议。

### 数据库 Schema（`decisionSessions` 表）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 主键 |
| userId | varchar | 用户 ID（可选，支持匿名） |
| dilemma | text | 用户的纠结描述 |
| clarifyQuestion | text | AI 追问内容（可选） |
| clarifyAnswer | text | 用户补充回答（可选） |
| answers | text | 5 道题的完整选择记录（JSON） |
| awakeningQuote | text | 点醒金句 |
| analysis | text | 潜意识分析 |
| tendency | varchar | 内心倾向标签 |
| actionAdvice | text | 行动建议 |
| createdAt | timestamp | 创建时间 |

1. **Freemium 基础订阅**：免费提供每日限次的测试体验；Pro 版解锁无限次测试、专属决策风格档案（MVP 雏形见 [ADR-006](../decisions/006-user-profile-prompt-injection.md)）。
