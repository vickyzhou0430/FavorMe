<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Flutter UI to ViewModel | User-provided raw question enters app state |
| Flutter client to backend API | Untrusted mobile request crosses network with bearer token and device id |
| Local config/storage to HTTP client | Development token, base URL, and stable device id are read into requests |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01-01 | Information Disclosure | `insight_api_client.dart` | mitigate | Keep AI/vendor keys and prompts out of Flutter; only use backend `API_TOKEN` via local configuration or dart-define, and grep for vendor key names. |
| T-02-01-02 | Spoofing | `device_id_store.dart` | mitigate | Generate and persist a non-empty stable development device id; always send `X-Device-Id` with business API requests. |
| T-02-01-03 | Information Disclosure | `insight_view_model.dart` | mitigate | Do not log raw question bodies; state may preserve input for UI only. |
| T-02-01-04 | Tampering | `insight_api_client.dart` | mitigate | Parse backend JSON into typed question models and surface malformed responses as typed errors rather than fabricating questions. |
</threat_model>
