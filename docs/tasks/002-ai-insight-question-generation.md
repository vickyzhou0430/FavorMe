# 任务 002：AI 心理洞察三问生成逻辑

## 目标

将「AI 心理洞察助手」最关键的能力沉淀为可实现、可调试、可迭代的文档规范，确保后端、客户端、提示词调优使用同一套约束。

核心交付：稳定输出三道高质量问题，帮助用户识别内在 `51%` 倾向，而不是替用户做决定。

## 背景与定位

- 产品从“外部答案提供者”转为“心理洞察引导工具”。
- 典型场景是生活中的“小而即时纠结”，强调轻量和效率。
- 交互形态为问卷式快答（3 问），不走冗长聊天流。

## MVP 范围

主链路：`用户提问 -> (可选)补充背景 -> 生成三问 -> 用户作答 -> 输出倾向结论`

本任务只定义「三问生成逻辑与约束」，不覆盖完整 UI 视觉、会员计费闭环、复杂内容审核平台。

## 输入定义

### 输入字段

- `raw_question`: 用户原始问题（文本或语音转文字）
- `input_mode`: `text` | `voice_to_text`
- `background_text`: 可选补充背景，允许为空
- `user_persona`: 用户基础画像（性别、年龄段、MBTI 等，MVP 可部分可空）

### 输入有效性校验

满足以下任一情况即拦截，不进入三问生成：
- 非问题句、无明确决策语义
- 纯噪声输入、长度过短且无可解析意图
- 违法或高风险内容（按安全策略拒答）

返回建议：`422 INVALID_QUESTION_INPUT`，并给用户可理解提示。

## 问题生成模式

### 场景分类（内部判断，不作为 API 字段输出）

三问生成前先判断用户纠结属于哪类，并按对应逻辑生成问题；分类只用于提示词内部推理，保持现有 API 响应结构兼容。

1. **无差别小事决策**  
   无关三观、无关未来、无风险，两个选项后果差异极小；纠结核心是缺乏启动动机或选择惰性。问题围绕当下状态、省力省心、低后悔成本，以及是否被“应该/自律”绑架。
2. **人生大事与三观选择**  
   影响长期人生路径，与自我认同和价值观强绑定；纠结核心是方向迷茫和未来恐惧。问题围绕真实向往、恐惧底线、是否逃避压力或改变成本。
3. **人际关系类纠结**  
   围绕他人、关系、社交展开；冲突是自我舒适与关系体面。问题围绕真在意还是怕冲突、是否委屈自己、抛开社交压力后的真实想法。
4. **小型自我冲突**  
   事情不大但拉扯明显，两个选项都有吸引力或阻力；纠结核心是看不清 51% 真实倾向。问题围绕真实期待、遗憾成本、是否被计划/成本/自律等标准绑架。

如果问题同时符合多类，优先选择最核心冲突：关系问题优先归入人际关系；长期路径优先归入人生大事；后果差异极小优先归入无差别小事；其余内心拉扯归入小型自我冲突。

## 三问框架（强约束）

每次必须输出且仅输出 3 题，每题带 2-4 个可选项（便于滑动或点击）。

1. **真实偏好 / 当下状态**  
   目的：识别更贴近自己的真实偏好、期待或当下舒服状态。
2. **后悔边界 / 恐惧底线**  
   目的：识别更不想承受的后悔、委屈、消耗或未来代价。
3. **真心 vs 绑架 / 逃避**  
   目的：识别倾向来自真心想要，还是外界标准、关系压力、计划成本、选择惰性或逃避。

禁止项：
- 直接给结论、替用户下命令
- 医疗诊断、精神治疗替代描述
- 算命断言、恐吓、迷信化表达

## 输出结构（建议）

```json
{
  "generation_mode": "generic",
  "questions": [
    {
      "id": "q1",
      "dimension": "inner_preference",
      "title": "如果只听从你当下的真实直觉，你更偏向哪一边？",
      "options": [
        { "id": "A", "label": "选项A描述" },
        { "id": "B", "label": "选项B描述" }
      ]
    }
  ]
}
```

要求：
- `dimension` 固定映射三类维度（`inner_preference` / `fear_boundary` / `active_vs_avoidance`）
- 文案短句化，避免多重从句
- 选项语义互斥、可快速判断
- 选项必须围绕用户给出的两个选择或隐含两端生成，避免抽象标签化

## 倾向结论生成规则

结论生成同样先内部判断场景分类，再根据用户三问答案输出一个自然段的 51% 倾向反馈。MVP 暂保持现有 API 结构：`{ conclusion: string }`。

要求：
- 表达“当下更偏向哪一侧”，不替用户做最终决定。
- 贴合场景核心冲突：小事强调省力省心和低后悔；人生大事强调真实向往、未来恐惧和逃避成本；人际关系强调真在意、委屈感和边界；小型自我冲突强调真实期待、遗憾成本和绑架因素。
- 不输出标题、列表或 JSON。
- 避免医疗诊断、算命断言、恐吓和命令式表达。

## 评分与倾向输出（MVP 建议）

- 对每个选项配置倾向权重（可在服务端配置表维护）。
- 三题汇总得到 `tendency_label` 与 `confidence_band`（例如 low/medium/high）。
- 输出分层：
  - 免费：倾向标签 + 一句话理由
  - 会员（预留）：更完整的结构化解析

## 观测与调试

建议记录：
- 问题类型、生成模式（generic/contextual）
- 三问内容与用户选择分布
- 输入拦截率、三问完成率、结果页停留时长
- 人工抽检标签（问题质量是否达标）

## 验收标准（UAT）

- 输入有效时，系统稳定返回 3 题且维度完整。
- 输入无效时，系统可解释性拦截，不生成三问。
- 用户能在短时间内完成作答（问卷式体验成立）。
- 结果能表达“倾向”而非“替你决定”，符合产品定位。
