# 产品范围（FavorMe）

> **状态：已拍板**（与 [`QUESTIONS.md`](QUESTIONS.md)、[`tech-stack.md`](tech-stack.md) 同步，2026-04-24）

## 1. 一句话

FavorMe 是面向年轻用户的**轻治愈 / 轻决策辅助**产品，通过**今日运势、心安指南、个人资料与收藏**等模块，以温和、可执行的方式帮助用户减少内耗、看见积极面。

## 2. 当前 `demo/web` 已具备的能力

- 手机号 + 固定验证码的**演示登录**（非生产短信）
- **新手引导**、**今日运势**、**心安指南**、**收藏/我的** 等  
- 数据**主要存浏览器 localStorage**，**用于产品/体验验证，不代表**最终商用数据、安全与合规方案

## 3. 首版（MVP / 7 月上市能力）**目标**（已拍板）

> 与「能卖钱」相比，**第一优先级**是打通 **可上架、可联调、可部署** 的全链路，并在此基础上**再**看商业数据。

### 3.1 全链路

- 中国大陆用户可访问的 **H5 内容**（薄壳 + `WebView`）+ **自建后端** + **阿里云**部署
- 真实 **鉴权/账号/用户资料/收藏与记录** 走**服务端**（下掉纯 localStorage 依赖，按接口渐进迁移即可）
- **大模型**经 **AI 网关** 调用，开发期**优先接火山**；**允许**境外 API，不强制数据不出境（后续**法务**单独立项）
- 应用**上架流程**在 **Android / iOS** 至少**跑通其一或双端**（以 7 月前实际资源为准，**技术路线已允许双端薄壳**）

### 3.2 首版**不做**（但预留扩展）

- **不** 上线**付费/会员/订阅**（不开放真实 IAP/微信支付宝等，除非单独变更决策）
- **不** 上**管理后台**、复杂运营/AB/投放体系（**商用第一版**再开）
- **不** 做**重**内容审核平台；**必须** 做 **MVP 级最小安全兜底**（见 [`tech-stack.md`](tech-stack.md) 第 6 节）

### 3.3 数据与商业化预留

- 在数据库设计中**预留** 订单/权益/订阅/用量 等扩展，避免日后付费改造推倒重来，但 **MVP 不实现支付闭环**

## 4. 市场与语言（已拍板）

- 首版：**中国大陆**
- 后续可：**海外**（时区/合规/模型/支付单独评估）

## 5. 客户端形态（与工程目录）

| 阶段 | 形态 | 说明 |
|------|------|------|
| MVP / 7 月首版 | 各端 **WebView 薄壳 + H5** | 最快速度全链路、上架、联调；H5 逐步换接真实 API |
| MVP 之后 | **Flutter** 单仓双端 | 目录 `clients/flutter/` 占位，见 [`tech-stack.md`](tech-stack.md) 与 [ADR-003](decisions/003-client-mvp-webview-flutter.md) |
| 旧 `clients/android` / `ios` | 薄壳工程**可**在此落地 | 与 Flutter 长期**并存过渡**或**迁移**策略实施时再定，不在 MVP 内纠结 |

**系统版本**：**近 2 个主要大版本**（在客户端工程初始化时写入具体 `min` 版本号）。

## 6. 明确不做的产品方向（除非决策变更）

- 医疗诊断、精神治疗替代
- 以「算命断言、恐吓、迷信话术」为卖点的高风险内容（**法务与内容策略**最终定稿）
- 将**厂商密钥、完整可商用 prompt 配置** 长期**仅**放在客户端
