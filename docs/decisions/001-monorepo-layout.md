# ADR-001：Monorepo 目录与文档优先

- **状态**：已接受
- **日期**：2026-04-24
- **决策者**：项目维护者（与 AI 协助初始化）

## 背景

FavorMe 需要从 **Web Demo** 演进到 **后端 + Android + iOS**，同时希望 Cursor 等工具能长期通过**仓库内文档**恢复上下文，而非依赖单条聊天。

## 决策

1. 采用 **单仓 Monorepo**，根目录下分 `demo/`、`backend/`、`clients/`，共享 `docs/` 与 `AGENTS.md`。
2. 将原 `favorme/` Next 应用迁移为 **`demo/web`**，作为演示与 UI/流程验证，不视为最终唯一交付形态。
3. 产品、架构、技术栈与任务以 **`docs/`** 为权威来源；重大技术选择补充 **ADR**。

## 备选方案

- **多仓**（demo、backend、各客户端分仓库）：隔离强，但跨仓文档与版本对齐成本高，首阶段不采用。
- **仅保留 demo 在根目录**：后续加 backend 时根目录会再次重组，迁移成本更高。

## 后果

- 正面：结构清晰，利于后续初始化 `backend` 与客户端工程；文档可接力。
- 负面：单仓体积会随子项目增长；需约定各子目录独立 `package`/`gradle` 等，避免混装依赖。
