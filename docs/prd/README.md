# PRD（产品需求文档）

本目录存放**可开发、可验收**的需求；高层商业边界以 [`../product-scope.md`](../product-scope.md) 为准，**不重复**抄「产品愿景」大段，而是拆成**可执行**的 PRD。

## 1. 命名与引用

- 文件：`NNN-模块或功能-短名.md`（`NNN` 三位，升序，全仓库**唯一**）。
- 引用方式：在任务单、Figma 描述、MR 中写 **「PRD-0NN」** 即可对应该文件号。

## 2. 索引

| 编号 | 文件 | 状态 | 对应 Figma/页面 | 与 demo 对应 `demo/web` 路径/路由（若有） | 产品负责人 | 最后更新 |
|------|------|------|----------------|----------------------------------------|------------|----------|
| — | （暂无） | — | — | — | — | — |

> 新建 PRD 时在此表**新增一行**；关闭或合并到其它 PRD 时把状态改为 **已合入/废弃** 并注明**替代**编号。

## 3. 新建

1. 复制 [`_template.md`](_template.md)。  
2. 取下一个**可用编号**（见上表或取当前最大 NNN+1）。  
3. 补全与 [`../design/figma-sources.md`](../design/figma-sources.md) 的链接。  
4. 若你改了 `demo/web`：**同 PR** 里更新本 PRD 的「与 demo 的对应」与「修订记录」。

## 4. 与 demo 的同步（产品主责）

- **推荐**：以 PRD 为验收依据，`demo` 是**快速可点**的验证，二者**行**为一致。  
- **强约束**（与 [`../team-workflow.md`](../team-workflow.md) 一致）：改 `demo/web` 而不改 PRD **不允许合入**（除非纯开发的技术重构且与产品无涉——由开发在 PR 中说明并 @ 产品）。
