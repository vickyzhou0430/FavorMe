## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ViewModel to backend submit API | Preserved question snapshot and selected answers cross the network |
| Result text to UI | Backend-generated conclusion is rendered in mobile UI |
| Reset action to local state | Round data is cleared from UI state at end of flow |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-02-01 | Tampering | `submitInsight` payload | mitigate | Submit exactly `rawQuestion`, complete `questions` snapshot, and selected `answers`; never rely on `sessionId`. |
| T-02-02-02 | Repudiation | `insight_api_client.dart` | mitigate | Preserve optional backend `requestId` in typed errors for Plan 03 diagnostics without logging raw question bodies. |
| T-02-02-03 | Information Disclosure | `result_card.dart` | accept | Conclusion is intentionally displayed to the user; do not persist it beyond in-memory round state in this phase. |
| T-02-02-04 | Denial of Service | `insight_view_model.dart` | mitigate | Guard question 3 completion so submit fires once while `submittingAnswers` disables actions. |
</threat_model>
